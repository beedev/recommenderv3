    Architecture Overview¶

    Multi-Strategy Search Pattern¶

    The configurator uses multiple complementary search strategies that are orchestrated in parallel or sequentially to provide comprehensive product search results.

    Available Strategies: - CypherSearchStrategy - Graph-based compatibility search using Neo4j relationships - LuceneSearchStrategy - Full-text search with relevance ranking using Neo4j Lucene indexes - VectorSearchStrategy - Semantic similarity search using OpenAI embeddings (text-embedding-3-large) - LLMSearchStrategy - Retrieve-then-rerank using LLM evaluation (GPT-4o-mini)

    async def _execute_parallel(
    self,
    strategies: List[SearchStrategy],
    component_type: str,
    user_message: str,
    master_parameters: Dict[str, Any],
    selected_components: Dict[str, Any],
    limit: int,
    offset: int
) -> List[Optional[StrategySearchResult]]:
    """
    Execute multiple strategies concurrently using asyncio.gather().

    Benefits:
    - Maximum performance (parallel I/O)
    - Timeouts per strategy (prevents hanging)
    - Exception isolation (one failure doesn't block others)

    Returns:
        List of results or None for failed strategies
    """
    tasks = []
    for strategy in strategies:
        # Wrap each strategy call with timeout protection
        task = asyncio.wait_for(
            strategy.search(
                component_type=component_type,
                user_message=user_message,
                master_parameters=master_parameters,
                selected_components=selected_components,
                limit=limit,
                offset=offset
            ),
            timeout=self.timeout  # 30 seconds default
        )
        tasks.append(task)

    # Execute all tasks concurrently
    results = await asyncio.gather(*tasks, return_exceptions=True)

    # Handle exceptions gracefully
    processed_results = []
    for i, result in enumerate(results):
        if isinstance(result, Exception):
            logger.error(
                f"❌ Strategy {strategies[i].get_name()} failed: {result}"
            )
            if self.fallback_on_error:
                processed_results.append(None)  # Skip failed strategy
            else:
                raise result  # Propagate error
        else:
            processed_results.append(result)

    return processed_results

    async def _execute_sequential(
    self,
    strategies: List[SearchStrategy],
    component_type: str,
    user_message: str,
    master_parameters: Dict[str, Any],
    selected_components: Dict[str, Any],
    limit: int,
    offset: int
) -> List[Optional[StrategySearchResult]]:
    """
    Execute strategies one at a time in order.

    Benefits:
    - Lower memory usage
    - Predictable resource consumption
    - Early exit on first success (optional)

    Returns:
        List of results or None for failed strategies
    """
    results = []

    for strategy in strategies:
        try:
            result = await asyncio.wait_for(
                strategy.search(
                    component_type=component_type,
                    user_message=user_message,
                    master_parameters=master_parameters,
                    selected_components=selected_components,
                    limit=limit,
                    offset=offset
                ),
                timeout=self.timeout
            )
            results.append(result)

        except Exception as e:
            logger.error(f"❌ Strategy {strategy.get_name()} failed: {e}")
            if self.fallback_on_error:
                results.append(None)
            else:
                raise

    return results

    async def search(
    self,
    component_type: str,
    user_message: str,
    master_parameters: Dict[str, Any],
    selected_components: Dict[str, Any],
    limit: int = 10,
    offset: int = 0
) -> Dict[str, Any]:
    """
    Execute multi-strategy search and consolidate results.

    Flow:
    1. Detect proactive vs user-intent mode
    2. Select context-appropriate strategies
    3. Execute strategies (parallel or sequential)
    4. Consolidate results with scoring
    5. Apply pagination
    6. Generate consolidation report

    Returns:
        Dict with products, scores, metadata, report
    """
    logger.info(
        f"🔍 SearchOrchestrator.search() | Component: {component_type} | "
        f"Mode: {self.execution_mode} | Message: '{user_message[:50]}...'"
    )

    # Step 1: Context-based strategy selection
    command_keywords = ["skip", "done", "next", "finalize", "yes", "no", ""]
    is_proactive_display = user_message.lower().strip() in command_keywords

    if is_proactive_display:
        context_strategies = ["cypher"]
        logger.info("🎯 PROACTIVE MODE: Using Cypher only (fast compatibility)")
    else:
        context_strategies = ["cypher", "lucene"]
        logger.info("🎯 USER INTENT MODE: Using Cypher + Lucene (full search)")

    # Step 2: Filter enabled strategies matching context
    enabled_strategies = [
        s for s in self.strategies
        if s.config.get("enabled", True) and s.get_name() in context_strategies
    ]

    if not enabled_strategies:
        logger.warning("⚠️ No enabled strategies found")
        return {
            "products": [],
            "scores": {},
            "metadata": {"error": "No strategies available"},
            "consolidation_report": {}
        }

    # Step 3: Execute strategies
    if self.execution_mode == "parallel":
        strategy_results = await self._execute_parallel(
            enabled_strategies, component_type, user_message,
            master_parameters, selected_components, limit, offset
        )
    else:
        strategy_results = await self._execute_sequential(
            enabled_strategies, component_type, user_message,
            master_parameters, selected_components, limit, offset
        )

    # Step 4: Filter successful results
    successful_results = [r for r in strategy_results if r is not None]

    if not successful_results:
        logger.warning("⚠️ All strategies returned no results")
        return {
            "products": [],
            "scores": {},
            "metadata": {"all_strategies_failed": True},
            "consolidation_report": {}
        }

    # Step 5: Consolidate results
    consolidated = self.consolidator.consolidate(
        strategy_results=[(r.strategy_name, r.products, r.scores)
                         for r in successful_results],
        master_parameters=master_parameters,
        component_type=component_type
    )

    # Step 6: Apply pagination
    paginated = consolidated[offset : offset + limit]

    # Step 7: Generate consolidation report
    report = self.consolidator.generate_consolidation_report(
        strategy_results=[(r.strategy_name, r.products, r.scores)
                         for r in successful_results],
        consolidated_results=consolidated
    )

    logger.info(
        f"✅ Search complete | Total: {len(consolidated)} | "
        f"Returned: {len(paginated)} | Strategies: {len(successful_results)}"
    )

    return {
        "products": [
            {
                "gin": r.gin,
                "name": r.name,
                "category": r.category,
                "description": r.description,
                "specifications": r.specifications,
                "score": r.consolidated_score
            }
            for r in paginated
        ],
        "scores": {r.gin: r.consolidated_score for r in paginated},
        "metadata": {
            "total_results": len(consolidated),
            "returned_results": len(paginated),
            "strategies_used": [s.get_name() for s in enabled_strategies],
            "execution_mode": self.execution_mode
        },
        "consolidation_report": report
    }

    class SearchStrategy(ABC):
    """
    Abstract base class for all search strategies.

    Implementations must provide:
    - search(): Execute product search with scoring
    - validate_compatibility(): Check product compatibility
    - get_name(): Return strategy identifier
    """

    def __init__(self, config: Dict[str, Any]):
        """
        Initialize strategy with configuration.

        Args:
            config: Strategy configuration dict
                - enabled: bool (default: True)
                - weight: float (default: 1.0)
                - custom strategy parameters
        """
        self.config = config
        self.enabled = config.get("enabled", True)
        self.weight = config.get("weight", 1.0)

    @abstractmethod
    async def search(
        self,
        component_type: str,
        user_message: str,
        master_parameters: Dict[str, Any],
        selected_components: Dict[str, Any],
        limit: int = 10,
        offset: int = 0
    ) -> StrategySearchResult:
        """
        Execute search and return results with scores.

        Returns:
            StrategySearchResult with products, scores, metadata
        """
        pass

    @abstractmethod
    async def validate_compatibility(
        self,
        product_gin: str,
        selected_components: Dict[str, Any]
    ) -> bool:
        """
        Validate if product is compatible with selected components.

        Returns:
            True if compatible, False otherwise
        """
        pass

    def get_name(self) -> str:
        """Return strategy identifier (e.g., 'cypher', 'lucene')"""
        return self.__class__.__name__.replace("SearchStrategy", "").lower()

    master_parameters: Dict[str, Any],
    selected_components: Dict[str, Any],
    limit: int = 10,
    offset: int = 0

    Execute Cypher-based graph search.

    Flow:
    1. Delegate to ComponentSearchService
    2. Convert relationship priorities to scores
    3. Return standardized results

        f"🔍 CypherSearchStrategy.search() | Component: {component_type}"

    component_key = self._to_component_key(component_type)

    # Step 1: Delegate to ComponentSearchService
    search_results = await self.product_search.component_service.search(
        component_type=component_key,
        master_parameters=master_parameters,
        selected_components=selected_components,
        limit=limit,
        offset=offset
    )

    # Step 2: Convert priority to confidence score
    products = []
    scores = {}

    for product in search_results.products:
        # Convert priority to normalized score (0.1 - 1.0)
        score = self._priority_to_score(product.priority)
        scores[product.gin] = score

        products.append({
            "gin": product.gin,
            "name": product.name,
            "category": product.category,
            "description": product.description,
            "specifications": product.specifications or {}
        })

    logger.info(
        f"✅ CypherSearchStrategy complete | Products: {len(products)} | "
        f"Score range: {min(scores.values()):.2f}-{max(scores.values()):.2f}"
    )

    return StrategySearchResult(
        products=products,
        scores=scores,
        metadata={
            "search_method": "cypher",
            "total_results": len(products),
            "scoring": {
                "method": "priority_normalized_linear",
                "max_priority": self.max_priority,
                "min_score": self.min_score

    def _priority_to_score(self, priority: Optional[int]) -> float:
    """
    Convert Neo4j relationship priority to normalized confidence score.

    Algorithm: Normalized Linear Scoring

    Formula:
        score = max(min_score, 1.0 - (priority - 1) / (max_priority - 1))

    Configuration:
        max_priority = 20 (default)
        min_score = 0.1 (default)
        default_priority = 1 (for PowerSource without relationships)

    Score Mapping Examples:
        priority=1  → score=1.00 (perfect match, highest priority)
        priority=2  → score=0.95 (excellent match)
        priority=5  → score=0.79 (very good match)
        priority=10 → score=0.53 (moderate match)
        priority=15 → score=0.26 (low match)
        priority=20 → score=0.10 (minimum match)
        priority>20 → score=0.10 (capped at minimum)
        priority=None → score=1.00 (PowerSource, no relationships)

    Rationale:
        - Lower priority = Better match (Neo4j convention)
        - Linear normalization maintains relative ordering
        - Min score prevents zero scores (ensures all results rankable)
        - Capping prevents negative scores for outliers

    Args:
        priority: Priority from COMPATIBLE_WITH relationship (lower = better)

    Returns:
        Confidence score between min_score (0.1) and 1.0
    """
    if priority is None:
        # PowerSource has no priority (first component in workflow)
        priority = self.default_priority  # 1

    # Cap priority at max to prevent negative scores
    if priority > self.max_priority:
        logger.warning(
            f"⚠️ Priority {priority} exceeds max {self.max_priority}, "
            f"capping to min_score {self.min_score}"
        )
        return self.min_score

    # Normalized Linear Scoring
    # score = 1.0 - (priority - 1) / (max_priority - 1)
    #
    # Example calculation for priority=10, max_priority=20:
    # score = 1.0 - (10 - 1) / (20 - 1)
    #       = 1.0 - 9 / 19
    #       = 1.0 - 0.474
    #       = 0.526
    normalized_score = 1.0 - (priority - 1) / (self.max_priority - 1)

    # Ensure score doesn't go below min_score
    final_score = max(self.min_score, normalized_score)

    return final_score

    async def validate_compatibility(
    self,
    product_gin: str,
    selected_components: Dict[str, Any]
) -> bool:
    """
    Validate product compatibility using graph relationships.

    Checks:
    - Product exists in Neo4j
    - COMPATIBLE_WITH relationships exist to all selected components

    Returns:
        True if compatible, False otherwise
    """
    try:
        # Use Neo4j compatibility check query
        query = """
        MATCH (product:Product {gin: $product_gin})
        MATCH (selected:Product)
        WHERE selected.gin IN $selected_gins
        MATCH (product)-[:COMPATIBLE_WITH]->(selected)
        WITH product, count(DISTINCT selected) as compatible_count
        WHERE compatible_count = $required_count
        RETURN product.gin as gin
        """

        selected_gins = [
            comp.get("gin") for comp in selected_components.values()
            if comp and comp.get("gin")
        ]

        if not selected_gins:
            return True  # No compatibility requirements

        result = await self.product_search.neo4j_driver.execute_query(
            query,
            product_gin=product_gin,
            selected_gins=selected_gins,
            required_count=len(selected_gins)
        )

        return len(result.records) > 0

    except Exception as e:
        logger.error(f"❌ Compatibility validation failed: {e}")
        return False

    def _extract_lucene_score(self, product) -> Optional[float]:
    """
    Extract Lucene relevance score from product.

    Score Location Options:
    1. In specifications dict: {"lucene_score": 0.85}
    2. Appended to name: "Product Name (Score: 11.5)"

    Score Range: 0.0 - unbounded (typically 0.1 - 20.0)

    Args:
        product: ProductResult object

    Returns:
        Lucene score or None if not found
    """
    try:
        # Option 1: Check specifications dict
        if hasattr(product, "specifications") and product.specifications:
            if "lucene_score" in product.specifications:
                score = float(product.specifications["lucene_score"])
                logger.debug(
                    f"📊 Extracted Lucene score from specifications | "
                    f"GIN: {product.gin} | Score: {score:.4f}"
                )
                return score

        # Option 2: Check name for appended score
        if hasattr(product, "name") and product.name:
            import re
            # Match pattern: "Product Name (Score: 11.5)"
            match = re.search(r'\(Score: (\d+\.\d+)\)', product.name)
            if match:
                score = float(match.group(1))
                logger.debug(
                    f"📊 Extracted Lucene score from name | "
                    f"GIN: {product.gin} | Score: {score:.4f}"
                )
                return score

        # No score found
        logger.debug(f"⚠️ No Lucene score found for GIN: {product.gin}")
        return None

    except Exception as e:
        logger.warning(
            f"⚠️ Could not extract Lucene score for GIN: {product.gin} | "
            f"Error: {e}"
        )
        return None

    Retrieval Methods:
    - lucene: Full-text search only
    - vector: Semantic search only
    - combined: Both methods with deduplication (default)
    """
    logger.info(f"LLM search for {component_type} using {self.retrieval_method} retrieval")

    all_candidates = []

    if self.retrieval_method == "lucene":
        # Lucene-only retrieval
        lucene_result = await self.lucene_strategy.search(
            component_type=component_type,
            user_message=user_message,
            master_parameters=master_parameters,
            selected_components=selected_components,
            limit=self.retrieval_limit,
            offset=0
        )
        all_candidates = lucene_result.products
        logger.info(f"Retrieved {len(all_candidates)} candidates from Lucene")

    elif self.retrieval_method == "vector":
        # Vector-only retrieval
        vector_result = await self.vector_strategy.search(
            component_type=component_type,
            user_message=user_message,
            master_parameters=master_parameters,
            selected_components=selected_components,
            limit=self.retrieval_limit,
            offset=0
        )
        all_candidates = vector_result.products
        logger.info(f"Retrieved {len(all_candidates)} candidates from Vector")

    elif self.retrieval_method == "combined":
        # Combined retrieval with deduplication
        logger.info(f"Running combined retrieval (Lucene + Vector)")

        # Parallel retrieval
        lucene_result, vector_result = await asyncio.gather(
            self.lucene_strategy.search(
                component_type=component_type,
                user_message=user_message,
                master_parameters=master_parameters,
                selected_components=selected_components,
                limit=self.retrieval_limit,
                offset=0
            ),
            self.vector_strategy.search(
                component_type=component_type,
                user_message=user_message,
                master_parameters=master_parameters,
                selected_components=selected_components,
                limit=self.retrieval_limit,
                offset=0
            )
        )

        # Deduplicate by GIN
        seen_gins = set()
        for product in lucene_result.products + vector_result.products:

    async def _rerank_with_llm(
    self,
    user_message: str,
    component_type: str,
    candidates: List[Dict[str, Any]],
    limit: int
) -> Tuple[List[Dict[str, Any]], Dict[str, float]]:
    """
    Use LLM to evaluate and rank candidates.

    Scoring Guidelines:
    - 90-100: Perfect match (exact competitor equivalent OR perfect spec match)
    - 70-89: Good match with minor gaps
    - 50-69: Partial match
    - 30-49: Weak match
    - 0-29: Poor match

    IMPORTANT: Products with matching competitor equivalents should score 90-100.
    """
    prompt = self._build_reranking_prompt(user_message, component_type, candidates)

    try:
        response = await self.openai_client.chat.completions.create(
            model=self.model,
            messages=[
                {
                    "role": "system",
                    "content": """You are an expert welding equipment specialist.

For each product, provide:
1. A relevance score (0-100) where:
   - 90-100: Perfect match (exact competitor equivalent OR perfect spec match)
   - 70-89: Good match with minor gaps
   - 50-69: Partial match
   - 30-49: Weak match
   - 0-29: Poor match

2. Brief reasoning (1-2 sentences)

IMPORTANT: When user asks for "equivalent of [Brand:Model]":
- Prioritize products with that Brand:Model in "Competitor Equivalents" field
- Products with matching competitor equivalents should score 90-100
- This is the PRIMARY matching criterion

Return valid JSON with this structure:
{
    "products": [
        {"gin": "...", "score": 90, "reasoning": "..."},
        ...
    ]
}"""
                },
                {"role": "user", "content": prompt}
            ],
            temperature=self.temperature,  # 0.1 for consistency
            response_format={"type": "json_object"}  # Enforce JSON response
        )

        # Parse JSON response
        result = json.loads(response.choices[0].message.content)
        evaluations = result.get("products", [])

        # Create GIN → score mapping
        scores_dict = {
            eval_item["gin"]: float(eval_item["score"]) / 100.0  # Normalize 0-100 to 0-1
            for eval_item in evaluations
        }

        # Sort candidates by LLM score
        ranked_candidates = []
        for candidate in candidates:
            gin = candidate.get("gin")
            if gin in scores_dict:
                score = scores_dict[gin]

                # Find reasoning from evaluations
                reasoning = next(
                    (e.get("reasoning", "") for e in evaluations if e["gin"] == gin),
                    ""
                )

                # Add LLM metadata to specifications
                if "specifications" not in candidate:
                    candidate["specifications"] = {}
                candidate["specifications"]["llm_score"] = score * 100  # Original 0-100
                candidate["specifications"]["llm_reasoning"] = reasoning

                ranked_candidates.append((candidate, score))

        # Sort by score descending
        ranked_candidates.sort(key=lambda x: x[1], reverse=True)

        # Return top N products
        top_products = [p for p, _ in ranked_candidates[:limit]]

        logger.info(f"LLM re-ranking complete: {len(top_products)} products ranked")

        return top_products, scores_dict

    except Exception as e:
        logger.error(f"LLM re-ranking failed: {e}")
        # Fallback: return candidates as-is
        return candidates[:limit], {}

    def add_compatibility_filters(
    self,
    query: str,
    params: Dict,
    component_type: str,
    selected_components: Dict,
    node_alias: str = "target",
    collect_parent_gins: bool = False
) -> Tuple[str, Dict, Optional[str]]:
    """
    Add COMPATIBLE_WITH relationship filters for selected components.

    Handles:
    - Single-select: Core components (PowerSource, Feeder, Cooler, etc.)
    - Multi-select: Accessories (can have multiple selected)

    Args:
        query: Base query string
        params: Query parameters dict
        component_type: Component key
        selected_components: Dict of selected components from ResponseJSON
        node_alias: Target node variable name
        collect_parent_gins: Whether to collect parent GINs for scoring

    Returns:
        Tuple of (updated_query, updated_params, parent_alias)
    """
    config = self.component_config.get(component_type)
    dependencies = config.get("dependencies", [])

    if not dependencies:
        logger.debug(
            f"🔧 No dependencies for {component_type}, skipping compatibility filters"
        )
        return query, params, None

    compatibility_clauses = []
    rel_counter = 1
    parent_alias = None

    for dep in dependencies:
        # Map dependency to ResponseJSON key
        # e.g., "power_source" → "PowerSource"
        response_key = self._to_response_key(dep)
        selected = selected_components.get(response_key)

        if not selected:
            logger.debug(f"⏭️ Skipping dependency {dep}, not selected")
            continue

        dep_config = self.component_config.get(dep)
        dep_label = dep_config["neo4j_label"]
        dep_alias = f"{dep[:2]}_dep"

        # Check if multi-select (list of components)
        if isinstance(selected, list):
            # Multi-select: Accessories
            gins = [item.get("gin") for item in selected if item.get("gin")]

            if not gins:
                continue

            # Build parameter names for each GIN
            param_names = []
            for idx, gin in enumerate(gins):
                param_name = f"{dep}_gin_{idx}"
                params[param_name] = gin
                param_names.append(f"${param_name}")

            # Add MATCH clauses
            compatibility_clauses.append(f"MATCH ({dep_alias}:{dep_label})")
            compatibility_clauses.append(
                f"WHERE {dep_alias}.gin IN [{', '.join(param_names)}]"
            )
            compatibility_clauses.append(
                f"MATCH ({node_alias})-[r{rel_counter}:COMPATIBLE_WITH]->({dep_alias})"
            )

            logger.debug(
                f"🔗 Added multi-select compatibility | Dependency: {dep} | "
                f"Count: {len(gins)}"
            )

        else:
            # Single-select: Core component
            gin = selected.get("gin")

            if not gin:
                continue

            param_name = f"{dep}_gin"
            params[param_name] = gin

            # Add MATCH clauses
            compatibility_clauses.append(
                f"MATCH ({dep_alias}:{dep_label} {{gin: ${param_name}}})"
            )
            compatibility_clauses.append(
                f"MATCH ({node_alias})-[r{rel_counter}:COMPATIBLE_WITH]->({dep_alias})"
            )

            if collect_parent_gins and parent_alias is None:
                parent_alias = dep_alias

            logger.debug(
                f"🔗 Added single-select compatibility | Dependency: {dep} | "
                f"GIN: {gin}"
            )

        rel_counter += 1

    # Combine all clauses
    if compatibility_clauses:
        query = "\n".join([query] + compatibility_clauses)

    return query, params, parent_alias

        params: Query parameters dict

        node_alias: Target node variable name

        Tuple of (updated_query, updated_params)
    """

    feature_terms = search_terms_dict.get("feature_terms", [])

    conditions = []

    # Product name filter (space-insensitive)
    if product_name:
        conditions.append(
            f"replace(toLower({node_alias}.item_name), ' ', '') CONTAINS "
            f"replace(toLower($product_name_filter), ' ', '')"
        )
        params["product_name_filter"] = product_name

        logger.debug(f"🔍 Added product name filter: '{product_name}'")

    # Feature term filters
    if feature_terms:
        for idx, term in enumerate(feature_terms):
            param_name = f"feature_{idx}"
            conditions.append(
                f"(toLower({node_alias}.clean_description) CONTAINS toLower(${param_name}) "
                f"OR toLower({node_alias}.attribute_ruleset) CONTAINS toLower(${param_name}))"
            )
            params[param_name] = term

            logger.debug(f"🔍 Added feature term filter [{idx}]: '{term}'")

    # Combine conditions with OR
    if conditions:
        where_clause = "\nWHERE (" + " OR ".join(conditions) + ")"
        query += where_clause

    return query, params

    def build_lucene_query(
    self,
    component_type: str,
    user_message: str,
    node_alias: str = "p"
) -> Tuple[str, Dict]:
    """
    Build Lucene full-text search query with UNION for better recall.

    Strategy:
    1. Remove stop words from user message
    2. Normalize text (units: 500 Amps → 500 A)
    3. Escape Lucene special characters
    4. Build UNION query if normalized != stopwords-removed

    UNION Benefits:
    - Query 1 (normalized): Better precision with standardized units
    - Query 2 (stopwords-removed): Better recall with original text

    Example Input:
        "I need a 500 Amps MIG welder"

    Example Processing:
        Stopwords removed: "500 Amps MIG welder"
        Normalized: "500 A MIG welder"

    Example Output Query:
        CALL {
            CALL db.index.fulltext.queryNodes('productIndex', '500 A MIG welder')
            YIELD node AS p, score
            WHERE p:PowerSource AND p.category = $category
            RETURN p, score
            UNION
            CALL db.index.fulltext.queryNodes('productIndex', '500 Amps MIG welder')
            YIELD node AS p, score
            WHERE p:PowerSource AND p.category = $category
            RETURN p, score
        }

    Args:
        component_type: Component key
        user_message: User's search text
        node_alias: Node variable name

    Returns:
        Tuple of (query_string, parameters_dict)
    """
    config = self.component_config.get(component_type)
    neo4j_label = config["neo4j_label"]
    category = config["category"]

    # Step 1: Remove stop words
    stopwords_removed = self._remove_stopwords(user_message)

    # Step 2: Normalize text (units, measurements)
    normalized_text = self._normalize_search_text(stopwords_removed)

    # Step 3: Escape Lucene special characters
    escaped_normalized = self._escape_lucene_special_chars(normalized_text)
    escaped_stopwords = self._escape_lucene_special_chars(stopwords_removed)

    logger.debug(
        f"📝 Lucene query preprocessing:\n"
        f"  Original: '{user_message}'\n"
        f"  Stopwords removed: '{stopwords_removed}'\n"
        f"  Normalized: '{normalized_text}'\n"
        f"  Escaped normalized: '{escaped_normalized}'\n"
        f"  Escaped stopwords: '{escaped_stopwords}'"
    )

    # Step 4: Build UNION query if different
    if normalized_text != stopwords_removed:
        # UNION: Normalized + Stopwords-removed for better recall
        query = f"""
        CALL {{
            CALL db.index.fulltext.queryNodes('productIndex', $normalized_text)
            YIELD node AS {node_alias}, score
            WHERE {node_alias}:{neo4j_label} AND {node_alias}.category = $category
            RETURN {node_alias}, score

            CALL db.index.fulltext.queryNodes('productIndex', $stopwords_text)
            YIELD node AS {node_alias}, score
            WHERE {node_alias}:{neo4j_label} AND {node_alias}.category = $category
            RETURN {node_alias}, score
        }}

        params = {
            "normalized_text": escaped_normalized,
            "stopwords_text": escaped_stopwords,
            "category": category
        }

        logger.debug(
            f"🔍 Built UNION Lucene query | "
            f"Normalized: '{escaped_normalized}' | "
            f"Stopwords: '{escaped_stopwords}'"
        )

    else:
        # Single query (no difference between normalized and stopwords-removed)
        query = f"""
        CALL db.index.fulltext.queryNodes('productIndex', $search_text)
        YIELD node AS {node_alias}, score
        WHERE {node_alias}:{neo4j_label} AND {node_alias}.category = $category

        params = {
            "search_text": escaped_stopwords,
            "category": category
        }

        logger.debug(
            f"🔍 Built single Lucene query | Text: '{escaped_stopwords}'"
        )

    return query, params

    def _normalize_parameter_value(
    self,
    value: str,
    component_type: str
) -> List[str]:
    """
    Normalize parameter values using JSON mappings.

    Returns all variants for a canonical value to match user input variations.

    Examples:
        >>> _normalize_parameter_value("5m", "interconnector")
        ["5m", "5 m", "5.0m", "5 meter", "5 meters", "5-m"]

        >>> _normalize_parameter_value("500A", "power_source")
        ["500A", "500 A", "500amp", "500 ampere", "500 amps", "500-A"]

        >>> _normalize_parameter_value("0.8mm", "feeder")
        ["0.8mm", "0.8 mm", "0.8millimeter", "0.8 millimeters"]

    Args:
        value: Input parameter value to normalize
        component_type: Component key (determines applicable normalizations)

    Returns:
        List of all variant strings for matching
    """
    if not self.normalizations:
        return [value.strip()]

    value_normalized = value.strip().lower()

    # Get applicable normalization types for this component
    applicable_params = self.component_parameter_mapping.get(
        component_type, []
    )

    # Check each normalization type
    for param_type in applicable_params:
        if param_type not in self.normalizations:
            continue

        mappings = self.normalizations[param_type].get("mappings", {})

        # Check if value matches any variant
        for canonical_value, variants in mappings.items():
            for variant in variants:
                if variant.lower() == value_normalized:
                    # Found match - return all variants
                    logger.debug(
                        f"✅ Normalized '{value}' → {len(variants)} variants | "
                        f"Type: {param_type}"
                    )
                    return variants

    # No mapping found - apply fallback normalization
    return self._fallback_normalization(value)

def _fallback_normalization(self, value: str) -> List[str]:
    """
    Fallback normalization when no JSON mapping exists.

    Generates common variations:
    - Original value
    - Lowercase
    - With spaces between number and unit
    - Without spaces

    Examples:
        >>> _fallback_normalization("500A")
        ["500A", "500a", "500 A", "500 a"]

        >>> _fallback_normalization("MIG welder")
        ["MIG welder", "mig welder"]

    Args:
        value: Input value

    Returns:
        List of normalized variants
    """
    variants = [value.strip()]

    # Add lowercase
    variants.append(value.strip().lower())

    # Add variations with/without spaces for number+unit patterns
    import re

    # Pattern: number followed by letters (e.g., "500A")
    match = re.match(r'^(\d+)([A-Za-z]+)$', value.strip())
    if match:
        num, unit = match.groups()
        # Add spaced version
        variants.append(f"{num} {unit}")
        variants.append(f"{num} {unit.lower()}")

    # Remove duplicates while preserving order
    seen = set()
    unique_variants = []
    for v in variants:
        if v not in seen:
            seen.add(v)
            unique_variants.append(v)

    logger.debug(
        f"🔄 Fallback normalization: '{value}' → {len(unique_variants)} variants"
    )

    return unique_variants

    def _normalize_search_text(self, text: str) -> str:
    """
    Normalize measurement units in search text.

    Normalization Rules:
    1. Add space between numbers and units: "500A" → "500 A"
    2. Amperage: "500 Amps/Ampere/Ampères" → "500 A"
    3. Voltage: "380 Volts/Voltios" → "380 V"
    4. Power (Watts): "500 Watts" → "500 W"
    5. Power (Kilowatts): "4 kilowatts" → "4 kW"
    6. Length (meters): "15 meters/metres" → "15 m"
    7. Length (millimeters): "30 millimeters" → "30 mm"
    8. Pressure: "5 Bar/BAR" → "5 bar"
    9. Flow rate: "7 l/min" → "7 l/minute"
    10. Phase: "3ph" → "3 phase"
    11. Hertz: "50Hz" → "50 Hz", "60 Hertz" → "60 Hz"
    12. Percentage: "60%" → "60 %"

    Examples:
        >>> _normalize_search_text("500 Amps MIG welder")
        "500 A MIG welder"

        >>> _normalize_search_text("380 Volts 30mm wire")
        "380 V 30 mm wire"

        >>> _normalize_search_text("I need a 15meters cable")
        "15 m cable"

    Args:
        text: User search text

    Returns:
        Normalized text with standardized units
    """
    normalized = text

    # Rule 1: Add space between numbers and units
    # "500A" → "500 A", "30mm" → "30 mm"
    normalized = re.sub(r'(\d+)([A-Za-z]+)', r'\1 \2', normalized)

    # Rule 2: Amperage normalization
    # "500 Amps" / "500 Ampere" / "500 Ampères" → "500 A"
    normalized = re.sub(
        r'(\d+)\s*(Amps?|Amperes?|Ampères?)\b',
        r'\1 A',
        normalized,
        flags=re.IGNORECASE
    )

    # Rule 3: Voltage normalization
    # "380 Volts" / "380 Voltios" → "380 V"
    normalized = re.sub(
        r'(\d+)\s*(Volts?|Voltios?)\b',
        r'\1 V',
        normalized,
        flags=re.IGNORECASE
    )

    # Rule 4: Power (Watts) normalization
    # "500 Watts" → "500 W"
    normalized = re.sub(
        r'(\d+)\s*Watts?\b',
        r'\1 W',
        normalized,
        flags=re.IGNORECASE
    )

    # Rule 5: Power (Kilowatts) normalization
    # "4 kilowatts" → "4 kW"
    normalized = re.sub(
        r'(\d+)\s*kilowatts?\b',
        r'\1 kW',
        normalized,
        flags=re.IGNORECASE
    )

    # Rule 6: Length (meters) normalization
    # "15 meters" / "15 metres" → "15 m"
    normalized = re.sub(
        r'(\d+)\s*(meters?|metres?)\b',
        r'\1 m',
        normalized,
        flags=re.IGNORECASE
    )

    # Rule 7: Length (millimeters) normalization
    # "30 millimeters" → "30 mm"
    normalized = re.sub(
        r'(\d+)\s*millimeters?\b',
        r'\1 mm',
        normalized,
        flags=re.IGNORECASE
    )

    # Rule 8: Pressure normalization
    # "5 Bar" / "5 BAR" → "5 bar"
    normalized = re.sub(
        r'(\d+)\s*BAR\b',
        r'\1 bar',
        normalized,
        flags=re.IGNORECASE
    )

    # Rule 9: Flow rate normalization
    # "7 l/min" → "7 l/minute"
    normalized = re.sub(
        r'(\d+)\s*l/min\b',
        r'\1 l/minute',
        normalized,
        flags=re.IGNORECASE
    )

    # Rule 10: Phase normalization
    # "3ph" → "3 phase"
    normalized = re.sub(
        r'(\d+)ph\b',
        r'\1 phase',
        normalized,
        flags=re.IGNORECASE
    )

    # Rule 11: Hertz normalization
    # "50Hz" → "50 Hz", "60 Hertz" → "60 Hz"
    normalized = re.sub(
        r'(\d+)\s*(Hz|Hertz)\b',
        r'\1 Hz',
        normalized,
        flags=re.IGNORECASE
    )

    # Rule 12: Percentage normalization
    # "60%" → "60 %"
    normalized = re.sub(
        r'(\d+)%',
        r'\1 %',
        normalized
    )

    # Clean up extra spaces
    normalized = re.sub(r'\s+', ' ', normalized).strip()

    logger.debug(
        f"📝 Search text normalization:\n"
        f"  Original: '{text}'\n"
        f"  Normalized: '{normalized}'"
    )

    return normalized

    def _remove_stopwords(self, text: str) -> str:
    """
    Remove stop words from search text.

    Benefits:
    - Focuses Lucene on meaningful keywords
    - Reduces noise in full-text matching
    - Improves precision for technical queries

    Examples:
        >>> _remove_stopwords("I need a 500A MIG welder")
        "500A MIG welder"

        >>> _remove_stopwords("Can you show me feeders for aluminum")
        "feeders aluminum"

        >>> _remove_stopwords("I want to find a machine for welding")
        "machine welding"

    Args:
        text: User search text

    Returns:
        Text with stop words removed
    """
    # Lowercase and clean whitespace
    clean = text.lower().strip()
    clean = re.sub(r'\s+', ' ', clean)

    # Split into words
    words = clean.split()

    # Filter out stop words and very short words
    keywords = [
        word for word in words
        if word not in STOP_WORDS and len(word) > 1
    ]

    # Fallback: If all words were stop words, keep words with length > 1
    if not keywords and words:
        keywords = [word for word in words if len(word) > 1]

    result = ' '.join(keywords)

    logger.debug(
        f"🧹 Stop word removal:\n"
        f"  Original: '{text}'\n"
        f"  Cleaned: '{result}'\n"
        f"  Removed: {len(words) - len(keywords)} words"
    )

    return result

    def consolidate(
    self,
    strategy_results: List[tuple[str, List[Dict], Optional[Dict[str, float]]]],
    master_parameters: Optional[Dict[str, Any]] = None,
    component_type: Optional[str] = None
) -> List[ConsolidatedResult]:
    """
    Consolidate results from multiple strategies with optional boosting.

    Steps:
    1. Deduplicate products by GIN
    2. Track which strategies found each product
    3. Collect scores from all strategies
    4. Calculate weighted average score
    5. Apply exact match boosting (if enabled)
    6. Append scores to product names (if enabled)
    7. Sort by consolidated score (descending)
    8. Apply score threshold filtering (if component specified)

    Args:
        strategy_results: List of (strategy_name, products, scores) tuples
        master_parameters: User parameters for exact match boosting
        component_type: Component key for threshold filtering

    Returns:
        Sorted list of ConsolidatedResult objects
    """
    logger.info(
        f"🔄 Consolidating results from {len(strategy_results)} strategies"
    )

    # Step 1: Deduplicate by GIN
    products_by_gin: Dict[str, ConsolidatedResult] = {}

    for strategy_name, products, scores in strategy_results:
        logger.debug(
            f"📊 Processing strategy: {strategy_name} | "
            f"Products: {len(products)} | "
            f"Has scores: {scores is not None}"
        )

        for product in products:
            gin = product.get("gin")

            if not gin:
                logger.warning(f"⚠️ Product missing GIN, skipping")
                continue

            # First occurrence: Store product data
            if gin not in products_by_gin:
                products_by_gin[gin] = ConsolidatedResult(
                    gin=gin,
                    name=product.get("name", "Unknown"),
                    category=product.get("category", ""),
                    description=product.get("description", ""),
                    specifications=product.get("specifications", {}),
                    found_by_strategies=[],
                    strategy_scores={},
                    consolidated_score=0.0
                )

            # Step 2: Track which strategies found this product
            products_by_gin[gin].found_by_strategies.append(strategy_name)

            # Step 3: Collect scores from this strategy
            if scores and gin in scores:
                strategy_score = scores[gin]
            else:
                strategy_score = self.default_score
                logger.debug(
                    f"⚠️ No score for GIN {gin} from {strategy_name}, "
                    f"using default {self.default_score}"
                )

            products_by_gin[gin].strategy_scores[strategy_name] = strategy_score

    logger.info(
        f"✅ Deduplicated {len(products_by_gin)} unique products"
    )

    # Step 4: Calculate weighted average scores
    for gin, result in products_by_gin.items():
        result.consolidated_score = self._calculate_weighted_score(
            result.strategy_scores
        )

    # Step 5: Apply exact match boosting (100x)
    if master_parameters and component_type:
        self._apply_exact_match_boosting(
            products_by_gin, master_parameters, component_type
        )

    # Step 6: Append scores to product names
    if self.append_score_to_name:
        self._append_scores_to_names(products_by_gin)

    # Step 7: Sort by consolidated score (descending)
    sorted_results = sorted(
        products_by_gin.values(),
        key=lambda x: x.consolidated_score,
        reverse=True
    )

    # Step 8: Apply score threshold filtering
    if component_type:
        sorted_results = self._apply_score_threshold(
            sorted_results, component_type
        )

    logger.info(
        f"✅ Consolidation complete | Total: {len(sorted_results)} | "
        f"Score range: {sorted_results[0].consolidated_score:.2f} - "
        f"{sorted_results[-1].consolidated_score:.2f}"
    )

    return sorted_results

    def _calculate_weighted_score(
    self,
    strategy_scores: Dict[str, float]
) -> float:
    """
    Calculate weighted average score from multiple strategies.

    Formula:
        weighted_score = (Σ score_i × weight_i) / (Σ weight_i)

    Example:
        strategy_scores = {"cypher": 0.8, "lucene": 0.6}
        strategy_weights = {"cypher": 0.4, "lucene": 0.6}

        weighted_sum = (0.8 × 0.4) + (0.6 × 0.6) = 0.32 + 0.36 = 0.68
        total_weight = 0.4 + 0.6 = 1.0
        final_score = 0.68 / 1.0 = 0.68

    Args:
        strategy_scores: Dict mapping strategy name to score

    Returns:
        Weighted average score
    """
    if not strategy_scores:
        return self.default_score

    weighted_sum = 0.0
    total_weight = 0.0

    for strategy_name, score in strategy_scores.items():
        weight = self.strategy_weights.get(strategy_name, 1.0)
        weighted_sum += score * weight
        total_weight += weight

    if total_weight == 0:
        return self.default_score

    final_score = weighted_sum / total_weight

    logger.debug(
        f"📊 Weighted score calculation | "
        f"Strategies: {list(strategy_scores.keys())} | "
        f"Weighted sum: {weighted_sum:.4f} | "
        f"Total weight: {total_weight:.4f} | "
        f"Final: {final_score:.4f}"
    )

    return final_score

    def _apply_exact_match_boosting(
    self,
    products_by_gin: Dict[str, ConsolidatedResult],
    master_parameters: Dict[str, Any],
    component_type: str
):
    """
    Apply 100x score boosting for exact product name matches.

    Matching Strategy:
    - Extract product_name from master_parameters for component
    - Compare with result names using space-insensitive matching
    - Boost score by exact_match_boost multiplier (default 100.0)

    Example:
        User searches for: "Aristo 500ix"
        Product name: "Aristo 500ix MIG Power Source"

        Space-insensitive match:
        "aristo500ix" in "aristo500ixmigpowersource" ✓

        Original score: 0.65
        Boosted score: 0.65 × 100.0 = 65.0

    Args:
        products_by_gin: Dict of consolidated results
        master_parameters: User parameters with product names
        component_type: Component key for parameter extraction
    """
    # Map component_type to master_parameters key
    component_key_map = {
        "PowerSource": "power_source",
        "Feeder": "feeder",
        "Cooler": "cooler",
        "Interconnector": "interconnector",
        "Torch": "torch",
        "Accessories": "accessories"
    }

    component_key = component_key_map.get(component_type)
    if not component_key:
        logger.debug(f"⏭️ No component key mapping for {component_type}")
        return

    # Extract product_name from parameters
    component_params = master_parameters.get(component_key, {})
    product_name = component_params.get("product_name", "").strip().lower()

    if not product_name:
        logger.debug(f"⏭️ No product_name in master_parameters for {component_type}")
        return

    logger.info(
        f"🎯 Applying exact match boosting | "
        f"Component: {component_type} | "
        f"Target: '{product_name}' | "
        f"Boost: {self.exact_match_boost}x"
    )

    # Check each product for exact match
    boosted_count = 0

    for gin, result in products_by_gin.items():
        result_name = result.name.strip().lower()

        # Space-insensitive matching
        result_name_no_spaces = result_name.replace(' ', '')
        product_name_no_spaces = product_name.replace(' ', '')

        # Check if exact match
        is_exact_match = (
            result_name_no_spaces == product_name_no_spaces or
            product_name_no_spaces in result_name_no_spaces
        )

        if is_exact_match:
            original_score = result.consolidated_score
            result.consolidated_score = original_score * self.exact_match_boost
            boosted_count += 1

            logger.info(
                f"✨ BOOSTED: '{result.name}' (GIN: {gin})\n"
                f"   Matched: '{product_name}'\n"
                f"   Score: {original_score:.4f} → {result.consolidated_score:.4f} "
                f"({self.exact_match_boost}x)"
            )

    if boosted_count > 0:
        logger.info(
            f"✅ Exact match boosting complete | Boosted: {boosted_count} products"
        )
    else:
        logger.info(
            f"ℹ️ No exact matches found for '{product_name}'"
        )

    def _apply_score_threshold(
    self,
    results: List[ConsolidatedResult],
    component_type: str
) -> List[ConsolidatedResult]:
    """
    Filter products by score threshold percentage.

    Algorithm:
    1. Get component-specific threshold from config (default: 25%)
    2. Calculate minimum score: top_score × (1 - threshold/100)
    3. Keep only products with score >= minimum

    Example:
        Top score: 10.0
        Threshold: 25%
        Min score: 10.0 × (1 - 0.25) = 10.0 × 0.75 = 7.5

        Keep products with score >= 7.5 (within 25% of top)

    Configuration:
        component_types.json → lucene_score_threshold_percent

    Args:
        results: Sorted list of consolidated results
        component_type: Component key for config lookup

    Returns:
        Filtered list of results
    """
    if not results:
        return results

    # Load component config
    component_key_map = {
        "PowerSource": "power_source",
        "Feeder": "feeder",
        "Cooler": "cooler",
        "Interconnector": "interconnector",
        "Torch": "torch",
        "Accessories": "accessories"
    }

    component_key = component_key_map.get(component_type)
    if not component_key:
        logger.debug(f"⏭️ No threshold filtering for {component_type}")
        return results

    # Get threshold from config
    component_config = load_component_config().get(component_key, {})
    threshold_percent = component_config.get(
        "lucene_score_threshold_percent", 25
    )

    # Calculate minimum score
    top_score = results[0].consolidated_score
    min_score = top_score * (1 - threshold_percent / 100.0)

    # Filter results
    filtered_results = [
        r for r in results
        if r.consolidated_score >= min_score
    ]

    filtered_count = len(results) - len(filtered_results)

    logger.info(
        f"🔍 THRESHOLD FILTERING:\n"
        f"   Component: {component_type}\n"
        f"   Top score: {top_score:.2f}\n"
        f"   Threshold: {threshold_percent}%\n"
        f"   Min score: {min_score:.2f}\n"
        f"   Filtered out: {filtered_count} products\n"
        f"   Kept: {len(filtered_results)}/{len(results)} products"
    )

    return filtered_results

    def generate_consolidation_report(
    self,
    strategy_results: List[tuple[str, List[Dict], Optional[Dict[str, float]]]],
    consolidated_results: List[ConsolidatedResult]
) -> Dict[str, Any]:
    """
    Generate consolidation summary report.

    Includes:
    - Total unique products
    - Products per strategy
    - Strategy coverage (% of results found by each)
    - Score statistics (min, max, mean)
    - Multi-strategy products (found by 2+ strategies)

    Returns:
        Dict with consolidation metrics
    """
    report = {
        "total_unique_products": len(consolidated_results),
        "strategies": {},
        "score_statistics": {},
        "multi_strategy_products": 0
    }

    # Strategy coverage
    for strategy_name, products, scores in strategy_results:
        report["strategies"][strategy_name] = {
            "product_count": len(products),
            "has_scores": scores is not None,
            "coverage_percent": (
                len(products) / len(consolidated_results) * 100
                if consolidated_results else 0
            )
        }

    # Score statistics
    if consolidated_results:
        scores = [r.consolidated_score for r in consolidated_results]
        report["score_statistics"] = {
            "min": min(scores),
            "max": max(scores),
            "mean": sum(scores) / len(scores)
        }

        # Multi-strategy products
        report["multi_strategy_products"] = sum(
            1 for r in consolidated_results
            if len(r.found_by_strategies) > 1
        )

    logger.info(
        f"📊 CONSOLIDATION REPORT:\n"
        f"   Unique products: {report['total_unique_products']}\n"
        f"   Multi-strategy: {report['multi_strategy_products']}\n"
        f"   Score range: {report['score_statistics'].get('min', 0):.2f} - "
        f"{report['score_statistics'].get('max', 0):.2f}"
    )

    return report

    Performance Metrics¶

    Typical Query Times: - Proactive mode (Cypher only): 50-100ms - User intent mode (Cypher + Lucene): 150-300ms - With exact match boosting: +10-20ms overhead - Parallel vs sequential: 40-60% faster in parallel

    Score Ranges: - Cypher: 0.1 - 1.0 (normalized) - Lucene: 0.1 - 20.0 (typical) - Consolidated: 0.0 - ∞ (depends on strategies) - After boosting: 0.0 - ∞ (exact matches ~100x)

    Filtering Impact: - Threshold filtering: Typically removes 10-40% of results - Compatibility filtering: Variable (depends on selected components) - Score threshold: Component-specific (25-30% default)

    Summary¶

    The ESAB Configurator's search system provides comprehensive, accurate product recommendations through:

    1. Multi-Strategy Architecture - Combines graph compatibility (Cypher) with full-text relevance (Lucene)
    2. Intelligent Orchestration - Context-aware strategy selection and parallel/sequential execution
    3. Advanced Query Building - Modular Cypher construction with comprehensive normalization
    4. Sophisticated Scoring - Priority-to-score conversion, weighted consolidation, exact match boosting
    5. Result Consolidation - Deduplication, score merging, threshold filtering
    6. Performance Optimization - Parallel execution, timeout protection, graceful fallback

    This architecture ensures users receive the most relevant products while maintaining fast response times and high reliability.
