    üèóÔ∏è Dynamic State Machine (Configuration-Driven)¶

    MOST IMPORTANT: This system uses a runtime-generated state machine - you can add S8, S9...SN states via JSON configuration without touching Python code.

    How It Works: 1. StateFactory Pattern (app/models/state_factory.py): - ConfiguratorState enum is dynamically generated at startup - States defined in config/component_types.json - init_configurator_state() called in main.py lifespan

    1. Processor Registry (app/services/orchestrator/state_processors.py):
    2. Automatic processor assignment: SingleComponentProcessor, MultiComponentProcessor, PowerSourceProcessor

    3. Registry pattern: StateProcessorRegistry.get_processor(state) ‚Üí returns appropriate handler

    4. Runtime Configuration:

    5. All prompts loaded from config/state_prompts.json
    6. Transitions defined in config/state_transitions.json
    7. Component applicability rules in config/component_applicability.json

    Architecture: 1. Correlation ID Middleware (app/middleware/logging_middleware.py): - Every request gets unique correlation_id (UUID) - Automatically injected into all logs during request lifecycle

    1. Session Context Middleware:
    2. session_id bound to structlog.contextvars

    3. All logs during session processing include session context

    4. Environment-Specific Rendering:

    5. Production: JSON output for ELK/Splunk/CloudWatch
    6. Development: Human-readable colored console output

    üîç Product Ranking & Relevance System¶

    ProductRanker (app/services/ranker/product_ranker.py) provides deterministic, explainable product ordering.

    Ranking Strategy (Multi-Criteria): 

    1
2
3
4
5
6
    def rank_products(products: List[ProductResult]) -> List[ProductResult]:
    return sorted(products, key=lambda p: (
        not p.is_default,              # 1. Default products first
        not has_exact_phrase_match(p), # 2. Exact name matches second
        p.name.lower()                 # 3. Alphabetical (determinism)
    ))

    Lucene Score Integration: - Lucene scores appended to product names: "Aristo 500ix (Score: 95.0)" - Score threshold filtering: keeps products within 25% of top score - Configurable per-component in search_config.json

    Why Deterministic Ranking: 1. Reproducibility: Same query ‚Üí same order (critical for testing) 2. Explainability: Clear precedence rules for business stakeholders 3. Extensibility: Easy to add telemetry, bundle frequency, pricing tiers

    üîó Integrated Components & Bundling¶

    Integrated Cooler Detection (state_orchestrator.py lines ~1950-2000):

    Logic: 1. PowerSource has integrated cooler ‚Üí Auto-set applicability.Cooler = "N" 2. Skip cooler_selection state ‚Üí Proceed to interconnector_selection 3. Bundle cooler in ResponseJSON ‚Üí Show as part of PowerSource
