    Date: 2025-01-09 Status: Investigated but NOT Implemented Version: 1.0

    Problem Statement¶

    User Issue: Query "I am satisfied with a maximum output of 300 Amps" returns ALL power sources (including 400A, 500A models) instead of only those ‚â§300A.

    Expected Behavior: Only show products matching the constraint (e.g., Renegade ES300i)

    Actual Behavior: All 6 power sources shown without filtering

    def _matches_feature_requirements(...):
    for num_spec in features.get("numeric_specs", []):

    Known Issues¶

    Issue 1: Multiple Sources of Truth¶

    Problem: Three files define feature names independently: 1. master_parameter_schema.json ‚Üí current_output 2. parameter_extractor.py (LLM prompt) ‚Üí current_rating 3. category_features_llm.json ‚Üí Current Output

    Impact: Difficult to maintain consistency

    Solution: See "Future Enhancements" below

    Issue 2: No Schema Validation¶

    Problem: validate_component_dict() function exists but is never called

    Impact: LLM can extract ANY keys, not just schema-defined features

    Issue 3: Score Threshold Too Aggressive (Separate Issue)¶

    Problem: Lucene score threshold (25%) can eliminate valid products before operator filtering

    Example: "max 300A" query: - Warrior 400i: Score 7.1982 (kept) - Renegade ES300i: Score 3.2751 (eliminated by threshold)

    Analysis: See docs/RENEGADE_ISSUE_ANALYSIS.md

    Solution: Either: 1. Disable score threshold for PowerSource (score_threshold_percent: 0) 2. Apply operator filter BEFORE score threshold 3. Lower threshold to 5-10%

    Status: Separate from operator filtering issue
