    Multilingual Lucene Search Fix¶

    Date: November 8, 2025 Version: 2.1 Status: âœ… Implemented

    Executive Summary¶

    Critical Bug Fixed: Non-English queries (Spanish, French, German, Portuguese, Italian, Swedish) were receiving 0 search results when searching against English-only Neo4j database via Lucene full-text search.

    Root Cause: Raw non-English user messages were passed directly to Lucene search without translation, resulting in failed matches against English database content.

    Solution: Combined LLM translation + parameter extraction in single API call, followed by stopword removal, to create optimized English queries for Lucene search.

    Impact: - âœ… Fixes critical multilingual bug (zero search results â†’ accurate matches) - âœ… Improves English search quality (removes noise words) - âœ… Zero breaking changes (uses existing _metadata pattern) - âœ… No performance cost (same LLM call) - âœ… No external dependencies (simple Python stopwords)

# NEW: Stopword removal method (lines 2191-2224)
def _remove_stopwords(self, text: str) -> str:
    """Remove common English stopwords from search query."""
    words = text.lower().split()
    filtered = [word for word in words if word not in self.ENGLISH_STOPWORDS]
    return ' '.join(filtered) if filtered else text
