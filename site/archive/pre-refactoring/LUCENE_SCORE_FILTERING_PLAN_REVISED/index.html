    Lucene Score Filtering Plan (REVISED)¶

    Date: 2025-11-07 Status: Planning Phase - REVISED Goal: Add score-based filtering (configurable top %) and append score to product name without adding new fields

    üìã Executive Summary¶

    Enhance Lucene search with: 1. ‚úÖ Configurable score filtering: Return only products within X% of top score (default 25%) 2. ‚úÖ Score in product name: Append score to name like "Aristo 500ix (Score: 95.0)" 3. ‚úÖ Zero schema changes: No new fields in ProductResult (keeps frontend unchanged) 4. ‚úÖ Fully configurable: Percentage threshold in search_config.json

    üéØ Requirements¶

    Requirement 1: Configurable Top % Score Filtering¶

    Business Logic: - If top product has score = 1.0 and threshold = 25%, return products with score >= 0.75 - If top product has score = 0.8 and threshold = 30%, return products with score >= 0.56 - Formula: min_score_threshold = top_score * (1 - threshold_percent/100) - Configurable threshold in JSON (default 25%)

    üèóÔ∏è Architecture Design¶

    Approach: Modify Name in Lucene Search Only¶

    Pros: - ‚úÖ Zero schema changes - ProductResult stays the same - ‚úÖ Frontend displays score automatically (in product name) - ‚úÖ No API changes needed - ‚úÖ Easy to implement and test - ‚úÖ Fully backward compatible

    Cons: - Product name is modified (includes score text) - Harder to parse if frontend needs raw score value later

    Decision: This is the simplest and safest approach per user requirements.

    async def search_power_source_lucene(
    self,
    user_message: str,
    limit: int = 10,
    offset: int = 0,
    min_score: float = 0.5
) -> SearchResults:
    """
    Full-text search with configurable score-based filtering and score display.

    Reads from search_config.json:
    - score_threshold_percent: Keep products within X% of top score
    - append_score_to_name: Add score to product name
    - score_decimal_places: Score formatting precision
    """
    logger.info(f"üîç Lucene full-text search for PowerSource: '{user_message[:100]}...'")

    # üÜï STEP 1: Load configuration
    from ..config.configuration_service import get_config_service
    try:
        config = get_config_service().get_config("search_config")
        lucene_config = config.get("lucene_search", {})
        score_threshold_percent = lucene_config.get("score_threshold_percent", 25)
        append_score_to_name = lucene_config.get("append_score_to_name", True)
        score_decimal_places = lucene_config.get("score_decimal_places", 1)
        logger.info(
            f"Lucene config: threshold={score_threshold_percent}%, "
            f"append_score={append_score_to_name}, decimals={score_decimal_places}"
        )
    except Exception as e:
        logger.warning(f"Failed to load Lucene config: {e}, using defaults")
        score_threshold_percent = 25
        append_score_to_name = True
        score_decimal_places = 1

    try:
        # Execute Neo4j Lucene query (existing code)
        query = """
        CALL db.index.fulltext.queryNodes("productIndex", $user_message)
        YIELD node, score
        WHERE node.category = 'Powersource' AND score >= $min_score
        RETURN DISTINCT
            node.gin as gin,
            node.item_name as name,
            node.category as category,
            node.clean_description as description,
            node.attributes_ruleset as specifications_json,
            node as specifications,
            score
        ORDER BY score DESC
        SKIP $offset
        LIMIT $query_limit
        """

        params = {
            "user_message": user_message,
            "min_score": min_score,
            "offset": offset,
            "query_limit": limit * 2  # Query more to account for score filtering
        }

        async with self.driver.session() as session:
            result = await session.run(query, params)
            records = await result.data()

            if not records:
                logger.info("No products found in Lucene search")
                return SearchResults(
                    products=[],
                    total_count=0,
                    filters_applied={
                        "search_method": "lucene_fulltext",
                        "user_message": user_message[:100],
                        "min_score": min_score,
                        "message": "no_results"
                    },
                    compatibility_validated=False,
                    offset=offset,
                    limit=limit,
                    has_more=False
                )

            # üÜï STEP 2: Get top score and calculate threshold
            top_score = records[0]["score"]
            # Formula: Keep products with score >= (top_score * (1 - threshold_percent/100))
            # Example: top=1.0, threshold=25% => keep score >= 1.0 * 0.75 = 0.75
            score_multiplier = 1.0 - (score_threshold_percent / 100.0)
            min_threshold = top_score * score_multiplier

            logger.info(
                f"Score filtering: top={top_score:.3f}, threshold={score_threshold_percent}%, "
                f"min_score_required={min_threshold:.3f}"
            )

            # üÜï STEP 3: Filter products by score threshold
            products = []
            products_filtered_out = 0

            for record in records:
                record_score = record["score"]

                # Check if product passes score threshold
                if record_score < min_threshold:
                    products_filtered_out += 1
                    logger.debug(
                        f"Filtered out: {record['name']} "
                        f"(score={record_score:.3f} < threshold={min_threshold:.3f})"
                    )
                    continue

                # Parse specifications
                specs = record.get("specifications", {})
                if hasattr(specs, "__dict__"):
                    specs = dict(specs)
                specs = self._clean_neo4j_types(specs)

                # üÜï STEP 4: Append score to product name (if enabled)
                product_name = record["name"]
                if append_score_to_name:
                    # Format score with specified decimal places
                    score_formatted = f"{record_score * 100:.{score_decimal_places}f}"
                    product_name = f"{product_name} (Score: {score_formatted})"

                product = ProductResult(
                    gin=record["gin"],
                    name=product_name,  # üÜï Modified name with score
                    category=record["category"],
                    description=record.get("description"),
                    specifications=specs
                )
                products.append(product)

                logger.debug(
                    f"‚úÖ Kept: {record['name']} "
                    f"(score={record_score:.3f}, formatted_name={product_name})"
                )

            # Log filtering summary
            total_products = len(records)
            kept_products = len(products)
            logger.info(
                f"Score filtering: kept {kept_products}/{total_products} products "
                f"({products_filtered_out} filtered out)"
            )

            # üÜï STEP 5: Apply pagination AFTER score filtering
            has_more = len(products) > limit
            if has_more:
                products = products[:limit]

            logger.info(
                f"‚úÖ Lucene search returned {len(products)} products "
                f"after score filtering (has_more: {has_more})"
            )

            return SearchResults(
                products=products,
                total_count=len(products),
                filters_applied={
                    "search_method": "lucene_fulltext",
                    "user_message": user_message[:100],
                    "min_score": min_score,
                    "top_score": top_score,
                    "score_threshold_percent": score_threshold_percent,
                    "score_threshold": min_threshold,
                    "products_before_filter": total_products,
                    "products_after_filter": kept_products,
                    "products_filtered_out": products_filtered_out,
                    "append_score_to_name": append_score_to_name
                },
                compatibility_validated=False,
                offset=offset,
                limit=limit,
                has_more=has_more
            )

    except Exception as e:
        logger.error(f"‚ùå Lucene full-text search failed: {e}")
        logger.error(f"Query: {query}")
        logger.error(f"Params: {params}")
        # Return empty results on error
        return SearchResults(
            products=[],
            total_count=0,
            filters_applied={
                "search_method": "lucene_fulltext",
                "error": str(e)
            },
            compatibility_validated=False,
            offset=offset,
            limit=limit,
            has_more=False
        )

    Scenario: 10% threshold filters out all but 1 product

    Risk 2: Score in Name Breaks Parsing¶

    Scenario: Frontend tries to parse product name for exact matching

    Mitigation: - Keep score format consistent: " (Score: XX.X)" - Easy to strip with regex: name.replace(/\s*\(Score: \d+\.?\d*\)$/, '')

    Detection: Frontend automated tests

    Recovery: Disable score display with append_score_to_name: false

    Risk 3: Performance Impact¶

    Mitigation: Filtering is O(n) in-memory operation (fast)

    Detection: Monitor response times, P95 latency

    Recovery: Disable score filtering with score_threshold_percent: 100
