    def _get_category_features(self, state: str) -> Optional[str]:
    """
    Get formatted feature guidance text for a category based on state
    Returns None if no features available

    Args:
        state: Current configurator state (e.g., "power_source_selection")

    Returns:
        Formatted feature guidance text ready for display
    """
    # Map state to category name
    state_to_category = {
        "power_source_selection": "Powersource",
        "feeder_selection": "Feeder",
        "cooler_selection": "Cooler",
        "interconnector_selection": "Interconnector",
        "torch_selection": "Torch",
        "connectivity_selection": "Connectivity",
        "feeder_wear_selection": "Feeder Wear",
        "feeder_accessory_selection": "Feeder Accessories",
        "powersource_accessory_selection": "Powersource Accessories",
        "remote_selection": "Remote"
        # Add more mappings as needed
    }

    category = state_to_category.get(state)
    if not category:
        return None

    # Get features for this category
    category_data = self.category_features.get(category)
    if not category_data:
        return None

    # Return pre-formatted guidance text
    return category_data.get("guidance", "")

    Implementation Details¶

    Feature Extraction Process¶

    The category_features_llm.json file was generated through a systematic LLM-powered analysis:

    Step 1: Product Data Collection - Query all products in each category from Neo4j - Extract descriptions, specifications, and attributes

    Step 2: LLM Feature Analysis - Feed product data to GPT-4 for pattern extraction - Identify common numeric specifications (ranges, units) - Identify categorical options (processes, cooling types) - Extract capabilities and key features

    Step 3: Guidance Text Generation - Format extracted features into user-friendly text - Add appropriate emojis for visual clarity - Structure information hierarchically

    Step 4: Manual Review & Refinement - Verify accuracy against actual product data - Adjust ranges and options for completeness - Test guidance text with real user queries

    Lessons Learned¶

    1. Strategic Information Placement Matters¶

    Discovery: Placement of feature guidance significantly impacts usability

    Finding: - ‚ùå Before products: Users skip it, want to see products first - ‚úÖ After products, before selection: Perfect timing for decision-making - ‚ùå After selection prompt: Too late, users already confused

    Takeaway: Information architecture is as important as information content

    2. Translation Token Limits Need Headroom¶

    Discovery: Tight token limits cause silent failures

    Finding: - 200 tokens: 15-20% truncation rate in non-English - 500 tokens: <1% truncation rate - 1000 tokens: 0% truncation but unnecessary cost

    Formula: optimal_limit = max_expected_length * 1.25

    Takeaway: Always add 25% buffer for translation token limits

    3. Pre-Computed Features Reduce Latency¶

    Discovery: Real-time feature extraction too slow
