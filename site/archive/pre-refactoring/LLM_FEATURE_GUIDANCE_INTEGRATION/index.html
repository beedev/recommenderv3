    LLM Feature Guidance Integration¶

    Overview¶

    What We Built: An intelligent user guidance system that automatically displays available product features, specifications, and ranges for each category in the configurator workflow. This helps users formulate better search queries by showing them what's actually available in the catalog.

    How It Works: Uses GPT-4 to analyze product descriptions and extract structured features, which are then automatically appended to state prompts to guide users.

    Architecture¶

    Components¶

    1. Feature Extraction (Offline)
    2. Script: docs/scripts/extract_features_llm.py
    3. Uses GPT-4 to analyze product descriptions
    4. Generates app/config/category_features_llm.json

    5. Run once per catalog update

    6. Feature Loading (Runtime)

    7. Message Generator loads features on startup
    8. Cached in memory for fast access

    9. No additional API calls during user sessions

    10. Feature Display (User-Facing)

    11. Automatically appended to state prompts
    12. Shows before translation (works in all 7 languages)
    13. Formatted with emojis for visual clarity

    Performance Impact¶

    Memory¶

    • File size: 29KB (negligible)
    • Loaded once on startup
    • Cached in memory for fast access

    Latency¶

    • No additional API calls during user sessions
    • Feature text appended to prompt (< 1ms)
    • Translation time unchanged (same prompt size as before)

    API Costs¶

    • Extraction (One-Time): ~$1-2 for all categories
    • Runtime: $0 (no additional OpenAI calls)

    Benefits¶

    1. Better User Guidance¶

    • Users see exactly what's available
    • No guessing about specs or options
    • Faster query formulation

    2. Reduced Support Burden¶

    • Fewer "what can I search for?" questions
    • Self-service feature discovery
    • Clear specification ranges

    3. Improved Search Success¶

    • Users specify correct values
    • More precise queries
    • Better search results

    4. Future-Proof¶

    • Easy to update when catalog changes
    • Multilingual by default
    • Extensible to new categories

    1

    # Re-run extraction
python docs/scripts/extract_features_llm.py

# Restart server
pkill -f uvicorn; uvicorn app.main:app --reload

    Issue: Features in Wrong Language¶

    Cause: Translation happens after feature append Expected: Features translated with rest of prompt Check: Verify MultilingualTranslator is working

    Issue: Extraction Fails¶

    Common Errors: 1. OpenAI API Key Missing: Set OPENAI_API_KEY in .env 2. Rate Limit: Wait and retry, or reduce categories 3. Neo4j Connection: Verify database is accessible
