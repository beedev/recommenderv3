    Redis + RedisInsight (ESAB Configurator)¶

    A production-friendly setup for:

    • Redis (as a cache + coordination layer) on a Linux VM
    • RedisInsight (browser UI) in Docker
    • Container → Host connectivity via Docker bridge (172.17.0.1)
    • Secure access patterns (SSH tunnel or restricted public access)
    • App .env template and quick client example
    • Ops commands & troubleshooting

    Server: Azure Linux VM — public IP 40.76.252.224 Redis: system service on the VM RedisInsight: Docker container on the VM

    0) Prereqs¶

    • Ubuntu/Debian Linux with sudo
    • Docker installed (sudo apt-get install -y docker.io && sudo systemctl enable --now docker)
    • UFW or Azure NSG to restrict inbound ports

    sudo apt-get update
sudo apt-get install -y redis-server

# Bind to loopback + docker bridge
sudo sed -i 's/^#\? *bind .*/bind 127.0.0.1 ::1 172.17.0.1/' /etc/redis/redis.conf

# Supervision
sudo sed -i 's/^#\?supervised .*/supervised systemd/' /etc/redis/redis.conf

# Auth (set a strong password)
sudo sed -i '/^requirepass /d' /etc/redis/redis.conf
echo 'requirepass <REDIS_PASSWORD>' | sudo tee -a /etc/redis/redis.conf

# Cache profile (optional)
echo 'save ""' | sudo tee -a /etc/redis/redis.conf
sudo sed -i 's/^appendonly .*/appendonly no/' /etc/redis/redis.conf
echo 'maxmemory 1gb' | sudo tee -a /etc/redis/redis.conf
echo 'maxmemory-policy allkeys-lru' | sudo tee -a /etc/redis/redis.conf

# Allow bridge clients (we bind only to private addresses)
sudo sed -i 's/^protected-mode .*/protected-mode no/' /etc/redis/redis.conf

sudo systemctl enable redis-server
sudo systemctl restart redis-server

# Canonical connection string (prefer this in code)
REDIS_URL='redis://default:<REDIS_PASSWORD>@127.0.0.1:6379/0'

# Fallbacks
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_DB=0
REDIS_USERNAME=default
REDIS_PASSWORD=<REDIS_PASSWORD>

# Client tuning
REDIS_POOL_MAX_CONNECTIONS=100
REDIS_SOCKET_TIMEOUT=5
REDIS_SOCKET_CONNECT_TIMEOUT=5

# Namespacing & TTLs
CACHE_PREFIX=esabcfg:
CACHE_TTL=3600
CACHE_TTL_SHORT=300
CACHE_TTL_LONG=86400
LOCK_PREFIX=esabcfg:lock:
LOCK_TTL=30

REDIS_HEALTHCHECK_ON_START=true
REDIS_HEALTHCHECK_KEY=esabcfg:health

    # app.py
import os, json
from fastapi import FastAPI
from redis.asyncio import Redis

r = Redis.from_url(os.getenv("REDIS_URL", "redis://127.0.0.1:6379/0"),
                   decode_responses=True)

app = FastAPI()

@app.on_event("startup")
async def on_startup():
    if os.getenv("REDIS_HEALTHCHECK_ON_START","true").lower()=="true":
        await r.set(os.getenv("REDIS_HEALTHCHECK_KEY","health"), "ok", ex=60)

@app.get("/items/{item_id}")
async def get_item(item_id: str):
    key = f"{os.getenv('CACHE_PREFIX','cache:')}item:{item_id}"
    if (cached := await r.get(key)):
        return {"source":"cache","data":json.loads(cached)}
    data = {"id": item_id, "value": "computed"}
    await r.set(key, json.dumps(data), ex=int(os.getenv("CACHE_TTL","3600")))
    return {"source":"fresh","data":data}

    3) Deploy RedisInsight (Docker)¶

    Bind UI to localhost (safe by default) and forward to it when needed:

    1
2
3
4

    sudo docker volume create redisinsight-data
sudo docker run -d --name redisinsight \
  -p 127.0.0.1:6380:5540 \
  -v redisinsight-data:/data \
  --restart unless-stopped \
  redis/redisinsight:latest
curl -f http://127.0.0.1:6380/api/health/  # -> {"status":"up"}

    Connect RedisInsight → Redis (inside the UI):

    • Host: 172.17.0.1
    • Port: 6379
    • Username: default (or viewer)
    • Password: <REDIS_PASSWORD> (or <VIEWER_PASS>)
    • (Optional) Logical DB: 0

    This works because Redis is listening on the host’s Docker bridge IP.

# /etc/nginx/sites-available/redisinsight
server {
  listen 443 ssl;
  server_name 40.76.252.224;
  ssl_certificate /etc/nginx/ssl/redisinsight.crt;
  ssl_certificate_key /etc/nginx/ssl/redisinsight.key;

  auth_basic "Restricted";
  auth_basic_user_file /etc/nginx/.redisinsight_htpasswd;

  location / {
    proxy_pass http://127.0.0.1:5540;
    proxy_set_header Host $host;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  }
}

# run RedisInsight bound to localhost
sudo docker rm -f redisinsight
sudo docker run -d --name redisinsight \
  -p 127.0.0.1:5540:5540 \
  -v redisinsight-data:/data \
  --restart unless-stopped redis/redisinsight:latest

sudo nginx -t && sudo systemctl reload nginx
sudo ufw allow 443/tcp
sudo ufw deny 6379/tcp

    (Use a real domain + Let’s Encrypt for proper TLS when possible.)

    Azure NSG reminder: In your VM’s NSG, do not allow 6379 from “Any”. Restrict 6380/443 to your IPs, or leave them closed if you use tunnels.

    6) Troubleshooting Quicklist¶

    • RedisInsight says “Could not connect to 127.0.0.1:6379” Containers see their own 127.0.0.1. Use host Docker bridge IP: 172.17.0.1.

    • AUTH failed: protected mode from container Ensure:

    • requirepass set and loaded (CONFIG GET requirepass not empty), or

    • protected-mode no with bindings limited to 127.0.0.1, ::1, 172.17.0.1.

    • Timeout from browser Check UI health (curl 127.0.0.1:6380/api/health/), container logs, and that your tunnel / firewall rule is correct.

    • Azure NSG blocks Open only the port you use for the UI (6380 or 443). Keep 6379 closed externally.

    Notes¶

    • Replace placeholders like <REDIS_PASSWORD>, <VIEWER_PASS>, <YOUR_PUBLIC_IP>.
    • Since passwords appeared in earlier logs, it’s wise to rotate them now.
    • Keep Redis (6379) private. Only the UI port (via tunnel or proxy) should be reachable from the internet.
