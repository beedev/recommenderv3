    Deployment Checklist¶

    Pre-deployment verification checklist for ESAB Recommender V2.

    Pre-Deployment¶

    Environment Preparation¶

    • Server Requirements Met
    • OS: Ubuntu 20.04+ or CentOS 8+
    • CPU: 4+ cores (2+ minimum)
    • RAM: 8GB (4GB minimum)

    • Disk: 50GB+ free space

    • Software Installed

    • Python 3.11+ (verify: python3 --version)
    • Docker & Docker Compose (if using Docker)
    • Git

    • curl, wget

    • Network Configuration

    • Firewall configured (ports 22, 80, 443, 8000)
    • Domain name configured (if using)
    • SSL certificates ready (production)

    Database Services¶

    • Neo4j
    • Service running and accessible
    • Connection URI configured
    • Credentials set

    • Test connection: cypher-shell -a <URI> "RETURN 1;"

    • PostgreSQL

    • Service running and accessible
    • Database pconfig created
    • Credentials set

    • Test connection: psql -h <HOST> -U <USER> -d pconfig -c "SELECT 1;"

    • Redis (optional but recommended)

    • Service running and accessible
    • Password configured
    • Test connection: redis-cli -h <HOST> -a <PASSWORD> PING

    API Keys and Secrets¶

    • OpenAI API Key
    • Valid API key obtained
    • Sufficient credits/quota

    • Test: curl https://api.openai.com/v1/models -H "Authorization: Bearer <KEY>"

    • Application Secrets

    • SECRET_KEY generated (32+ characters random)
    • JWT_SECRET_KEY generated (32+ characters random)

    • Generate: python -c "import secrets; print(secrets.token_urlsafe(32))"

    • LangSmith (optional)

    • API key obtained (if using observability)
    • Project created

    Configuration Files¶

    • .env File
    • Copied from template
    • All required variables set
    • Passwords changed from defaults
    • Database connection strings updated

    • File permissions: chmod 600 .env

    • Security Review

    • No secrets in version control
    • Strong passwords used (20+ characters)
    • Different passwords for each service
    • CORS_ORIGINS set to production domains only

    Deployment¶

    Docker Deployment¶

    • Files Prepared
    • docker-compose.yml reviewed
    • .env file in src/backend/

    • Volumes configured for persistence

    • Deploy

    • Run: docker-compose up -d
    • Check status: docker-compose ps

    • All services "Up" and healthy

    • Initialization

    • Wait 60 seconds for startup
    • Check logs: docker-compose logs backend
    • No errors in logs

    Systemd Deployment¶

    • Files Transferred
    • Source code on server
    • Deployment scripts executable

    • Ownership correct: chown -R azureuser:azureuser /home/azureuser/esab_recommender-bh

    • Environment Setup

    • Virtual environment created
    • Dependencies installed: pip install -r requirements.txt

    • .env file configured

    • Service Installation

    • Service files copied to /etc/systemd/system/
    • systemd reloaded: systemctl daemon-reload

    • Services enabled: systemctl enable esab-recommender.target

    • Start Services

    • Run: systemctl start esab-recommender.target
    • Check status: systemctl status esab-recommender.service
    • Service "active (running)"

    Post-Deployment Verification¶

    Health Checks¶

    • Application Health
    • HTTP: curl http://localhost:8000/health
    • Response: {"status": "healthy", ...}

    • All databases "connected"

    • API Documentation

    • Access: http://localhost:8000/docs
    • Swagger UI loads correctly

    • All endpoints visible

    • Test Interfaces

    • Main UI: http://localhost:8000/static/index.html
    • Test configurator: http://localhost:8000/static/test_configurator.html
    • No console errors in browser

    Functional Tests¶

    • API Test

2
3
      curl -X POST http://localhost:8000/api/v1/configurator/message \
  -H "Content-Type: application/json" \
  -d '{"message": "I need a 500A MIG welder", "language": "en"}'
    • Receives JSON response
    • Contains product recommendations

    • No errors

    • Session Management

    • Create session via API
    • Session ID returned

    • Session persists across requests

    • Database Verification

    • Neo4j has product data: cypher-shell "MATCH (n) RETURN count(n);"
    • PostgreSQL archival working
    • Redis caching working (if enabled)

    Performance Verification¶

    • Response Times
    • Health check < 500ms
    • API requests < 3 seconds

    • LLM responses reasonable (5-10s)

    • Resource Usage

    • CPU usage < 50% idle
    • Memory usage < 4GB

    • Disk space sufficient (> 20GB free)

    • Concurrent Users

    • Test with 5-10 concurrent requests
    • No timeouts or errors
    • Response times acceptable

    Security Verification¶

    • Firewall
    • Only required ports open
    • Database ports not externally accessible

    • SSH key-based auth only (no passwords)

    • SSL/TLS (production)

    • Certificate installed and valid
    • HTTPS working
    • HTTP redirects to HTTPS

    • No mixed content warnings

    • Application Security

    • Debug mode disabled: DEBUG=false
    • Error details hidden in production
    • CORS properly configured
    • Rate limiting enabled

    Logging and Monitoring¶

    • Logs Accessible
    • Backend logs: tail -f /home/azureuser/esab_recommender-bh/logs/esab-recommender.log
    • No critical errors

    • Log rotation configured

    • Monitoring Setup (if applicable)

    • LangSmith traces visible
    • Error tracking configured (Sentry)
    • APM configured (New Relic)
    • Alerts configured

    Backup and Recovery¶

    • Backup System
    • Backup script tested: ./deployment/database/backups/backup.sh
    • Backup created successfully

    • Backup stored securely

    • Restore Test (staging only)

    • Restore script tested
    • Databases restored correctly

    • Application functional after restore

    • Backup Schedule

    • Cron job or automated backup configured
    • Retention policy set
    • Off-site backup configured

    Production-Specific¶

    Reverse Proxy (Recommended)¶

    • Nginx
    • Nginx installed and configured
    • Proxy to backend configured
    • SSL termination working
    • Static file caching configured
    • Gzip compression enabled

    High Availability (Optional)¶

    • Load Balancer
    • Multiple backend instances running
    • Load balancer configured
    • Health checks working

    • Session affinity configured (if needed)

    • Database Replication

    • Neo4j cluster/replication (if using)
    • PostgreSQL replication (if using)
    • Redis replication/cluster (if using)

    Documentation¶

    • Team Documentation
    • Deployment runbook accessible
    • Operations team trained
    • On-call rotation configured

    • Escalation procedures documented

    • Configuration Management

    • All config files in version control
    • .env.example updated
    • Deployment scripts tested
    • Rollback procedure documented

    Go-Live Checklist¶

    Pre-Launch¶

    • Final Testing
    • All test cases passed
    • Load testing completed
    • Security scan completed

    • User acceptance testing done

    • Team Readiness

    • Operations team briefed
    • Support team trained
    • On-call schedule active
    • Incident response plan ready

    Launch¶

    • Deploy to Production
    • Code deployed
    • Databases migrated
    • Services started

    • Health checks green

    • Smoke Tests

    • End-to-end workflow tested
    • All integrations working
    • No errors in logs

    Post-Launch¶

    • Monitoring
    • Dashboards checked
    • No critical errors
    • Performance metrics normal

    • User activity normal

    • 24-Hour Check

    • System stable for 24 hours
    • No critical issues
    • Backup completed successfully
    • Team debriefing scheduled

    Rollback Plan¶

    In case of issues:

    • Rollback Procedure Known
    • Previous version tagged
    • Database backup available
    • Rollback script ready

    • Rollback tested in staging

    • Rollback Triggers

    • Critical errors defined
    • Performance degradation thresholds set
    • Decision makers identified
