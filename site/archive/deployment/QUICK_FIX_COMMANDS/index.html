    # 1. Navigate to project directory
cd ~/project/ayna-pod-recommender

# 2. Backup current Dockerfile
cp deployment/docker/Dockerfile deployment/docker/Dockerfile.backup

# 3. Download the fixed Dockerfile from outputs to the docker directory
# (Replace the path with where you downloaded the fixed Dockerfile)
cp /path/to/downloads/Dockerfile deployment/docker/Dockerfile

# 4. Navigate to docker directory
cd deployment/docker

# 5. Stop current containers
sudo docker compose -f docker-compose.prod.yml down

# 6. Remove old image (force rebuild)
sudo docker rmi esab-recommender:latest

# 7. Rebuild with no cache
sudo docker compose -f docker-compose.prod.yml build --no-cache

# 8. Start services
sudo docker compose -f docker-compose.prod.yml up -d

# 9. Watch logs to verify success
sudo docker compose -f docker-compose.prod.yml logs -f backend

    üí° Quick Troubleshooting¶

    Issue Command
    Check if uvicorn exists in container sudo docker exec esab-backend-prod which uvicorn
    Check PATH in container sudo docker exec esab-backend-prod bash -c 'echo $PATH'
    Check user in container sudo docker exec esab-backend-prod whoami
    List installed packages sudo docker exec esab-backend-prod pip list
    Check file ownership sudo docker exec esab-backend-prod ls -la /home/appuser/.local/bin/

    üìû Still Having Issues?¶

    1. Check if you're using the correct Dockerfile path
    2. Ensure you ran --no-cache during rebuild
    3. Verify the Dockerfile changes were actually made
    4. Check Docker logs for other errors: sudo docker compose -f docker-compose.prod.yml logs
    5. Review the full FIX_PERMISSION_ERROR.md guide
