    Docker Compose Production Update Summary¶

    Key Changes Made¶

    1. Added Local Redis Service¶

    • Why: Production benefits from a local Redis instance for caching, similar to development
    • Configuration:
    • 1GB memory limit (vs 512MB in dev)
    • Persistence enabled with RDB snapshots
    • Password-protected
    • Resource limits defined

    2. Added RedisInsight Service¶

    • Purpose: Optional Redis monitoring and management UI
    • Access: Port 8001 (configurable via REDISINSIGHT_PORT)
    • Benefits: Real-time monitoring of cache performance in production

    3. Improved Redis Health Checks¶

    • Updated to use redis-cli --raw incr ping (matches dev setup)
    • More reliable health check mechanism
    • Proper start period defined

    4. Named Volumes¶

    • Consistent naming: esab-redis-data-prod, esab-redisinsight-data-prod
    • Easier volume management and backup
    • Clear separation from development volumes

    5. Enhanced Redis Configuration¶

    • Persistence: AOF + RDB snapshots for data durability
    • Memory: 1GB max memory (production-appropriate)
    • Eviction: LRU policy for cache management
    • Snapshots: Regular saves (900s/1 key, 300s/10 keys, 60s/10000 keys)

    6. Service Dependencies¶

    • Backend now properly waits for Redis health check
    • depends_on with condition: service_healthy

    7. Logging Configuration¶

    • Added logging for Redis and RedisInsight
    • Consistent log rotation policies
    • Production-appropriate log retention

    8. Resource Limits¶

    • Redis: 0.5 CPU / 1.5GB RAM (limit), 0.25 CPU / 512MB (reservation)
    • RedisInsight: 0.5 CPU / 512MB RAM
    • Backend: 2.0 CPU / 4GB RAM (limit), 1.0 CPU / 2GB (reservation)

    1

    1
    docker-compose -f docker-compose.prod.yml down -v

    Production Checklist¶

    • Set strong REDIS_PASSWORD in environment
    • Configure cloud database credentials in .env
    • Set secure SECRET_KEY and JWT_SECRET_KEY
    • Review and adjust resource limits based on your infrastructure
    • Configure firewall rules (expose only necessary ports)
    • Set up SSL/TLS termination (use reverse proxy like Nginx)
    • Configure backup strategy for Redis data
    • Set up monitoring and alerting
    • Review log retention policies
    • Test health checks and restart policies

    Differences from Development Setup¶

    Feature Development Production
    Dockerfile Dockerfile.dev Dockerfile
    Hot Reload ✅ Enabled ❌ Disabled
    Debug Mode DEBUG=True DEBUG=False
    Restart Policy unless-stopped always
    Redis Memory 512MB 1GB
    Resource Limits None Defined
    Logging Basic Rotation configured
    Volume Mounting Source code mounted No code mounting
    Redis Password Dev password Should be changed

    Security Recommendations¶

    1. Change Default Passwords: Update REDIS_PASSWORD from default
    2. Use Secrets Management: Consider Docker secrets or external vaults
    3. Network Isolation: Use private networks where possible
    4. SSL/TLS: Enable for all external connections
    5. Access Control: Restrict RedisInsight access (consider removing in high-security environments)
    6. Regular Updates: Keep base images updated
    7. Monitoring: Set up alerts for unusual activity

    Performance Issues¶

    • Monitor Redis memory usage in RedisInsight
    • Check resource limits: docker stats
    • Review logs for errors or warnings
    • Adjust CACHE_TTL if needed
    • Scale up resource limits if consistently hitting limits
